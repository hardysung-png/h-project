## 1. 개요

가족/지인 소규모 모임을 카카오톡 단톡방 대신 웹앱으로 관리한다.

핵심 가치는 세 가지다: (1) 여러 일정 분리 응답, (2) 참석 현황 한눈에 집계, (3) 어른/아이 차등 N빵 정산. 구글폼·Doodle·Lu.ma 어디에도 없는, 한국 가족 모임의 현실적 계산을 기본으로 제공한다.

동시에 Next.js 15 + Supabase 실전 학습의 결과물이기도 하다(vibe coding). 쓸모 있고, 배울 수 있고, 실제로 내가 쓰는 도구 하나를 완성한다.

---

## 2. 문제 정의

| 문제 | 지금 어떻게 하나 | 왜 고통스러운가 |
|------|----------------|----------------|
| 여러 일정 참석 응답 | 단톡방에 "저 점심은 O, 차담은 X요" | 누가 어느 일정에 오는지 위로 스크롤하며 수동 파악 |
| 참석 인원 집계 | 매번 손으로 수를 센다 | 숫자가 계속 바뀌고 실수가 난다 |
| 차등 N빵 계산 | 손 계산 or 엑셀 | 어른/아이 차등, 호스트 제외 등 규칙이 많아 매번 헷갈림 |

## 3. 대상 사용자

**호스트 (이벤트 주최자)**
- 가족 모임, 생일 파티, 소규모 여행 등을 기획하는 사람
- 스마트폰에 익숙한 30~50대
- 회원가입 후 이벤트를 만들고, 초대 링크를 단톡방에 공유한다

**참석자 (초대받은 가족/지인)**
- 링크를 받아 닉네임만 입력하면 끝
- 앱 설치 불필요, 회원가입 불필요
- 이미 응답했어도 같은 링크로 재제출해 수정 가능

## 4. V0.1 스코프

### 포함

| 기능 | 설명 |
|------|------|
| 이벤트 생성 | 제목 + 세부 일정 최대 3개(이름 + 날짜/시간) |
| 초대 링크 | 이벤트마다 고유 URL 자동 생성, 복사 버튼 제공 |

| 정산 제외 설정 | 참석자별로 정산 제외/포함 토글 |

## 5. 사용자 흐름

### 호스트

```
회원가입/로그인
  → "+ 새 이벤트" 클릭
  → 이벤트 제목 + 일정 입력 (예: "점심 모임" 9월 13일 12시, "차담회" 9월 13일 15시)
  → 저장 → 이벤트 상세 페이지 진입
  → "초대 링크 복사" → 가족 단톡방에 붙여넣기
  → 비용 항목 추가 (식비 15만, 장소비 5만, 케이크 3만)
  → 참석자가 모이면 → 호스트 본인 "정산 제외" 토글
  → 차등 N빵 결과 확인
```

### 참석자

```
카톡방에서 링크 클릭
  → 초대 페이지 (로그인 없음)
  → 닉네임 입력 + 어른/아이 선택
  → 일정별 참석/불참 선택
  → "응답 제출"
  → 완료 페이지 ("응답됐어요. 수정하려면 같은 이름으로 다시 제출하세요.")
```

## 6. 핵심 기능 상세

### 6-1. 이벤트 생성

- 이벤트 제목: 필수, 텍스트
- 세부 일정: 최대 3개. 각 일정에 이름(필수) + 날짜/시간(선택)
- 생성 즉시 고유 `invite_token` 자동 발급

### 6-2. 초대 링크

- 형태: `https://[도메인]/i/[invite_token]`
- 클릭 한 번으로 클립보드 복사
- 링크 접근에 로그인 불필요

### 6-3. 참석 응답

- 닉네임: 자유 입력, 필수
- 구분: 어른 / 아이 라디오
- 각 일정별 참석 / 불참 라디오
- 제출 시 닉네임 기준으로 기존 응답 덮어쓰기 (수정 지원)

### 6-4. 참석 집계 테이블

호스트 전용. 행 = 참석자, 열 = 일정. 셀에 O / X 표시.

```
이름       | 점심 모임 | 차담회
---------- | --------- | ------
엄마 (어른) |    O      |   O
아들 (어른) |    O      |   X
막내조카(아이)|   O      |   O
삼촌 (어른) |    X      |   O
```

하단에 일정별 참석 인원 합계 표시 (예: 점심 3/4명, 차담 3/4명).

### 6-5. 차등 N빵 정산

**계산 방식**

```
총액 = sum(비용 항목)

가중치(참석자) =
  0           if 정산 제외
  1           if 어른
  0.5         if 아이  (기본값, 변경 가능)

1인분 = 총액 / sum(모든 가중치)
내 몫 = 1인분 × 내 가중치

반올림 오차 → 마지막 포함 참석자에게 보정
```

**예시**

- 총액: 230,000원 (식비 150,000 + 장소비 50,000 + 케이크 30,000)
- 참석자: 엄마(어른), 아들(어른), 막내조카(아이), 호스트(제외)
- 가중치: 1 + 1 + 0.5 + 0 = 2.5
- 1인분: 230,000 / 2.5 = 92,000원
- 막내조카: 92,000 × 0.5 = 46,000원

**정산 결과 테이블**

```
이름         | 구분 | 인분   | 금액      | 제외
------------ | ---- | ------ | --------- | ----
엄마         | 어른 |  1.0   | 92,000원  |
아들         | 어른 |  1.0   | 92,000원  |
막내조카     | 아이 |  0.5   | 46,000원  |
호스트       | 어른 |  0.0   | 0원       | [제외]
```

## 7. 비기능 요구사항

| 항목 | 목표 |
|------|------|
| 성능 | 초대 페이지 LCP < 2초 (모바일 4G 기준) |
| 규모 | 이벤트당 최대 50명, 동시 접속 수십 명 이내 |
| 보안 | 호스트 기능은 Supabase Auth 인증 필수 |
| 보안 (V0.1 tradeoff) | 참석자 응답은 invite_token + 닉네임 매칭으로 관리. V0.2에서 참석자 개인 토큰으로 강화 예정 |
| 반응형 | 모바일 우선 설계 (카카오톡 링크 클릭 → 모바일 브라우저) |
| 언어 | 한국어 전용 |

## 9. 미해결 질문 (V0.2 결정 필요)

1. **응답 수정 보안**: 현재 닉네임 동일 = 덮어쓰기. 악의적 덮어쓰기 가능 → V0.2에서 참석자별 signed token 도입 검토
2. **호스트 정산 제외 기본값**: UI에서 호스트를 기본 "제외"로 설정할지 여부
3. **일정별 정산**: 일정마다 다른 참석자가 있을 때 비용도 일정별로 나눌지 (V0.2)
4. **실시간 반영**: 참석 응답 실시간 표시 (현재는 새로고침 필요, Supabase Realtime으로 V0.2 대응)
